    • Introduction

    Purpose

    This is The MaBOr project, created by Patricia Safriuc and Matei Buliga.

    Intended Audience

    Our intended audience include every person over the age of 16.

    Product Scope

    This project intendes to help people register and attend various Masquerade Balls. The site is very easy to use and explicitly written such that everybody may have an easy time navigating and finding the perfect event for them to attend.

    References

    The project was given to us by our university professor. It can be found on the floowing link, at the bottom of the page: https://profs.info.uaic.ro/~andrei.panu/webprojects.html

    Overall Description

    Product Perspective

    This project is a self-contained product, therefore it perfectly stands on its own.

    Product Functions

    The main functions of this product are the searching and viewing various events, buying tickets for attending and sending invitations. In addition to these, users may also leave comments and feedback and share them on social media.
    For admin users, there is also the function of adding a new event, setting all the information from music to restrictions.

    User Classes and Characteristics

    There is only 1 class: Clients. Clients is a class which give a full information about a client and their account such as name, surname, password, address, phone. So different clients will never be mixed in tables and they can be initialized by all these characteristics.

    Operating Environment

    This product does not deppend on the on-use operating system and its version.

    External Interface Requirements

    The website will be intuitive, easy to use and to read. It will be able to scale dynamically regarding of the screen size and resolution or the device that it has been accessed on.

    User Interfaces

    A top nav-bar will facilitate the navigation throughout the website. Under the "Events" tab, there will be an anditional vertical navigation bar, that will display the different categories of the events and a way to quickly access them and read more about them.

    Software Interfaces

    Databases will be used to store data about users accounts, as well as, the already booked seats for every event.

    Communication Interfaces

    E-mail will be a required feature to be used in the process of sharing an event that the user will attend to. Also Google Chrome will be the basic required browser for our website.

    System Features

    Login

    The first feature is the Login one. This is necessary for buying tickets, sending invitations and leaving comments. Unlogged users are only able to access the pages of various events and get the information provided there.
    In addition, the admin needs to be logged-in in order to be able to add a new event or to modify an existing one.

    View Events

    This feature does not require having an account. Every person interested may enter the main page of events and, then, choose from the side menu the event they may be interested in finding out about.

    Book Tickets

    This feature requires an account. After finding the perfect event, the user may buy their on-line tickets.

    Send Invitations

    This feature requires an account. After finding the perfect event, the user may send an invitation to other acquintances. The invitation is personalized and is sent via e-mail.

    Add Comments and Feedback

    This feature requires an account. The user may add comments during the event they have a ticket for, or give feedback after the event ended.

    Share on Social media

    This feature requires an account. The user may share the event they are attending on their social media.

    Other Nonfunctional Requirements

    Security Requirements

    For a user to access the website's full functionality, they will need an account. The passwords will be stored in a secured database along with every user's username.

    Software Quaity Attributes

    Adaptability, maintainability and reliability will be the core attributes of our website. We want to deliver an easy-to-use, reliable and efficient product to our users.
